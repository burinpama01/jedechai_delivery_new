2. ตารางประเมินความสอดคล้อง (SC Concrete PUA Rules Compliance)

ด้านล่างนี้คือผลการเปรียบเทียบระหว่างกฎเหล็ก 9 ข้อของ SC Concrete PUA กับสถานะการพัฒนาใน Codebase ปัจจุบัน:

กฎข้อกำหนด (Rule) สถานะ (Status) รายละเอียดข้อวิเคราะห์ / คำแนะนำ
1. โทนสีหลัก (น้ำเงิน-แดง) ผ่าน (Pass) มีการผูกโทนสีหลักของระบบตามที่ลูกค้ากำหนด ทั้งในการออกแบบ UI/UX และ Dashboard ของแอดมิน
2. Responsive & Mobile-PWA กึ่งผ่าน (Partial) หน้าจอเว็บแอปปรับเข้าหา Mobile-responsive แต่ตัว PWA ของ Driver App ยังคงรันเป็นแอป Flutter ธรรมดา ไม่ได้คอมไพล์เป็น PWA เว็บแท้
3. การอนุมัติออเดอร์โดยผู้จัดการ ไม่พบโค้ด (Missing) ระบบยังไม่มีการจำกัดหรือตรวจสอบปริมาณออเดอร์ (น้อย/มาก) และการแบ่งสิทธิ์ให้ ผู้จัดการ (Manager) อนุมัติออเดอร์ขนาดใหญ่แยกต่างหาก
4. Credit Limit ลูกค้า (0 - 100k) ไม่พบโค้ด (Missing) ยังไม่มีการจำกัดวงเงิน Credit Limit ของลูกค้าในฐานข้อมูล และสิทธิ์การอนุมัติวงเงินเครดิตยังไม่ได้ถูกมอบให้เฉพาะ ผู้จัดการ (Manager)
5. โครงสร้างราคาแบบ Multi-tier ไม่พบโค้ด (Missing) ระบบคำนวณราคาปัจจุบันยังไม่รองรับ Multi-tier ตามระยะทาง ตามจำนวน และราคาพิเศษเฉพาะรายลูกค้า (Customer-specific pricing)
6. Stock Alert แจ้งเตือนขั้นต่ำ ไม่พบโค้ด (Missing) ไม่พบตารางจัดการคลังสินค้า (Inventory) หรือระบบแจ้งเตือนเมือสต็อกสินค้าลดลงต่ำกว่าระดับความปลอดภัยขั้นต่ำ
7. LINE Notification แทน Email กึ่งผ่าน (Partial) มีระบบ AdminLineNotificationService ส่งข้อความเข้า LINE แต่ยังมีโค้ดหลายจุดเรียกใช้ send-admin-email (เช่น ตอนเติมเงิน และปุ่มแอดมิน) ซึ่งขัดกับกฎห้ามใช้อีเมล
8. Payroll Slip ห้ามผ่าน LINE ผ่าน (Pass) ระบบสลิปโอนเงิน (Transfer Slip) ทำงานผ่านการอัปโหลดไฟล์ในระบบตรง ไม่มีการส่งสลิปเงินเดือนหรือข้อมูลส่วนตัวผ่าน LINE
9. แจ้งเตือนธุรกิจที่สำคัญ กึ่งผ่าน (Partial) มีการผูกเรื่องออเดอร์เข้าใหม่ และการขอลบเข้า LINE notification แต่ยังขาดการแจ้งเตือนเครดิตเกิน, สินค้าใกล้หมด และลูกหนี้เกินกำหนด

3. ประเด็นความเสี่ยงและช่องว่างระบบที่สำคัญ (Gaps & Issues)

จากการ Audit ด้านความปลอดภัยและกฎระเบียบธุรกิจ พบประเด็นความปลอดภัย 3 จุดหลักที่ต้องปรับปรุงก่อนปล่อยระบบจริง:

🚨 อีเมลรั่วไหลและขัดกับนโยบาย LINE (Non-compliant Email Notifications)
รายละเอียด: โค้ดใน wallet_topup_screen.dart:511 และ settingsActionsBridge.js:353 มีการเรียกใช้ Edge Function send-admin-email เมื่อคนขับทำรายการเติมเงินและเมื่อตั้งค่าระบบ ซึ่งขัดกับความต้องการของลูกค้าที่ "ไม่ใช้อีเมล และใช้เฉพาะ LINE เท่านั้น"
แนวทางแก้ไข: ปรับปรุงโค้ดเหล่านี้ให้หันมาเรียกใช้ AdminLineNotificationService.notify ร่วมกับ Edge Function send-admin-line แทนทั้งหมด
⚠️ สิทธิ์แอดมินรั่วไหล (Lack of Role Separation - Manager vs Staff)
รายละเอียด: ในระบบปัจจุบัน (โดยเฉพาะใน Web Admin และ RPC ในฐานข้อมูล) ฟังก์ชันการทำงานที่วิกฤต เช่น การจัดการเงินกระเป๋า (Wallet Top-up/Withdrawal) และการอนุมัติจัดการบัญชี ยังคงเปิดสิทธิ์ให้บัญชีที่มี role = 'admin' ทำได้ทุกคน โดยยังไม่มีการแยกย่อยสิทธิ์พิเศษสำหรับ ผู้จัดการ (Manager) เพื่อดูแลเรื่องวงเงินเครดิตและอนุมัติใบสั่งซื้อขนาดใหญ่ตามข้อกำหนด SC Concrete
แนวทางแก้ไข: เพิ่มฟิลด์ is_manager หรือเพิ่มบทบาท manager ในตารางสิทธิ์ และสร้าง Policy ตรวจสอบในการเข้าถึงข้อมูลสำคัญ
💡 บล็อกเกอร์ทางเทคนิค (CLI Deployment Lock)
รายละเอียด: โค้ด Schema การปฏิเสธขอลบบัญชีแบบ Atomic ( migrations sql ) ยังไม่ได้ถูกดันขึ้น Production จริง เนื่องจาก Supabase CLI ใน Sandbox ไม่มี Access Token
แนวทางแก้ไข: ผู้พัฒนาหลักต้องล็อกอิน Supabase CLI เพื่อสั่ง supabase db push หรือติดตั้ง SQL ผ่านแดชบอร์ดโดยตรง

4. แผนปฏิบัติงานเสนอแนะ (Proposed Roadmap to Compliance)

เพื่อนำระบบ Jedechai Delivery ให้บรรลุข้อกำหนดและกติกาธุรกิจของ SC Concrete PUA ทางเราขอเสนอแผนการพัฒนา 3 ระยะหลักดังนี้:

ระยะที่ 1: กำจัดระบบอีเมล และเปลี่ยนผ่านเข้าสู่ LINE Notification ทั้งระบบ

  • งานที่ 1: ปรับเปลี่ยนการเติมเงินใน wallet_topup_screen.dart จาก send-admin-email เป็น send-admin-line
  • งานที่ 2: ลบหรือระงับการทำงานของ Edge Function send-admin-email เพื่อความปลอดภัย
  • งานที่ 3: ผูกการแจ้งเตือนเพิ่มเติม เช่น สถานะสินค้าใกล้หมด และเตือนเครดิตลูกหนี้เกินกำหนดเข้ากลุ่ม LINE

ระยะที่ 2: ปรับปรุงโครงสร้างตารางข้อมูลเพื่อรองรับสิทธิ์ผู้จัดการ (Manager) และ Credit Limit

  • งานที่ 1: สร้าง SQL Migration อัปเกรดตาราง profiles ให้รองรับบทบาท manager หรือเพิ่มสิทธิ์ผู้จัดการ
  • งานที่ 2: เพิ่มฟิลด์ credit_limit และ current_credit_usage ในตารางลูกค้า
  • งานที่ 3: เขียน Database RLS Policy ป้องกันไม่ให้พนักงานทั่วไป (Staff) แก้ไขหรืออนุมัติ Credit Limit ได้นอกจาก Manager

ระยะที่ 3: ระบบคำนวณราคาพิเศษ คลังสินค้าขั้นต่ำ และจัดทำ PWA

  • งานที่ 1: เพิ่มระบบตรวจสอบสต็อกสินค้าและยิงแจ้งเตือน LINE ทันทีเมือต่ำกว่าขั้นต่ำ (Stock Alert)
  • งานที่ 2: พัฒนาระบบคำนวณราคาขนส่งแบบ Multi-tier ตามสเปกของ SC Concrete
  • งานที่ 3: ปรับแต่ง Driver App ให้รองรับการทำงานในรูปแบบ PWA อย่างสมบูรณ์แบบ
