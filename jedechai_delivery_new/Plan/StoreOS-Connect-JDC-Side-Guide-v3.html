StoreOS Connect - JDC Side Guide v3

สถานะ: ปรับ scope ล่าสุด 2026-06-30 ตาม decision ใหม่จากผู้ใช้

Decision ล่าสุด

  • ไม่ออก key แยกรายร้าน: JDC มี StoreOS connection ชุดเดียวทั้งระบบ
  • StoreOS เป็นฝั่งผูกร้าน: StoreOS ใช้ merchant_id ที่ร้าน copy จากแอป JDC ไป map ร้านเอง
  • Admin-web: ออก JDC key และ webhook secret ระดับระบบเท่านั้น
  • Merchant app: แสดง merchant_id เพื่อ copy ไปใส่ StoreOS

Contract StoreOS -> JDC

ทุก request ยังใช้ system credential เดียวกันใน headers:

X-JDC-Connection-Key: <system JDC key>
X-Connect-Event-Id: <unique event id>
X-Connect-Timestamp: <epoch seconds>
X-Connect-Signature: sha256=<HMAC raw body with webhook secret>

ค่า timestamp ใช้ epoch seconds หรือ epoch milliseconds ภายในช่วงเวลา clock skew ที่ JDC กำหนด ไม่ใช่ ISO string

และต้องใส่ merchant_id ใน JSON body เพื่อระบุร้าน JDC:

{
  "merchant_id": "JDC merchant UUID",
  "...": "payload per endpoint"
}

Verification Required

  • admin-web ออก key/secret โดยไม่ต้องกรอก merchant_id
  • StoreOS ยิง endpoint โดยส่ง merchant_id ใน body แล้ว JDC sync เข้าร้านนั้น
  • request ที่ไม่มี merchant_id ต้องถูก reject
  • outbound webhook จาก JDC ยังส่ง merchant_id ให้ StoreOS ใช้ map ร้าน
