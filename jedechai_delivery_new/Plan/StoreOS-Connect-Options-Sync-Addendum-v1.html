StoreOS Connect connect_upsert_menu 2026-07-02

เอกสารเพิ่มเติมจาก StoreOS-Connect-JDC-Side-Guide-v2.html — ฝั่ง StoreOS ส่ง field ใหม่ option_groups มากับทุก item ใน payload ของ connect_upsert_menu แล้ว ฝั่ง JDC ต้องอัปเดต Edge Function ให้ upsert ตัวเลือกลงตาราง menu_option_groups / menu_options / menu_item_option_links

1) Payload ใหม่ (ฝั่ง StoreOS ส่งแล้ว — backward compatible)

POST {functions_base}/connect_upsert_menu
Headers: X-JDC-Connection-Key: <jdc_xxx>          // connection key ที่ JDC admin ออก
         X-Connect-Event-Id: <uuid>                // กัน replay (JDC เก็บ event id ที่รับแล้ว)
         X-Connect-Signature: sha256=<hex hmac_sha256(raw_body, webhook_secret)>
         X-Connect-Timestamp: <epoch วินาที>

{
  "merchant_id": "<uuid ร้านฝั่ง JDC>",
  "full_sync": false,
  "items": [
    {
      "external_ref": "<products.id ฝั่ง StoreOS>",
      "name": "ข้าวกะเพรา",
      "description": "…",
      "price": 65,
      "image_url": null,
      "is_available": true,
      "category": "อาหาร",
      "preparation_time": null,
      "option_groups": [                       // ★ ใหม่ (#12)
        {
          "name": "ตัวเลือก",                  // จาก product_variants (บังคับเลือก 1)
          "min_selection": 1,
          "max_selection": 1,
          "options": [
            { "name": "ธรรมดา", "price": 0,  "is_available": true },
            { "name": "พิเศษ",  "price": 10, "is_available": true }
          ]
        },
        {
          "name": "ท็อปปิ้ง",                  // จาก modifier_groups
          "min_selection": 0,
          "max_selection": 3,
          "options": [
            { "name": "ไข่ดาว", "price": 10, "is_available": true }
          ]
        }
      ]
    }
  ]
}
เงื่อนไขที่ StoreOS ใช้สร้าง option_groups:
  • สินค้ามี variant ที่ active ≥ 2 ตัว → ส่งเป็นกลุ่มชื่อ "ตัวเลือก" (min 1 / max 1); มี 0–1 ตัว = ไม่ส่ง
  • modifier group แบบ single → max_selection = 1; แบบ multiple → ตาม max_selections ของร้าน
  • กลุ่มบังคับ (is_required) → min_selection ≥ 1
  • ตัด option/variant ที่ inactive ออกก่อนส่ง; กลุ่มไม่มี option เหลือ = ไม่ส่งทั้งกลุ่ม
  • price เป็นส่วนเพิ่มจากราคาเมนู (บาท, ทศนิยมได้ — JDC menu_options.price เป็น int ให้ round ฝั่ง JDC)

2) สิ่งที่ JDC ต้องทำใน Edge Function connect_upsert_menu ✅ IMPLEMENTED 2026-07-02

ทำแล้ว (2026-07-02): migration 20260702150000 (menu_option_groups.source + ย้าย order.created ไป trigger บน booking_items พร้อม items จริง + ts เป็นตัวเลข) + connect_upsert_menu รองรับ option_groups แบบ replace ต่อเมนู (source='storeos', merchant_id=null → ร้านแก้ใน JDC ไม่ได้ตามดีไซน์)
ขั้นรายละเอียด
1. เตรียม schemaเพิ่มคอลัมน์ menu_option_groups.external_source text (เช่น 'storeos') หรือใช้ naming convention เพื่อแยกกลุ่มที่มาจาก sync ออกจากกลุ่มที่ร้านสร้างเองใน JDC — กัน sync ไปลบของที่ร้านทำมือ
2. Upsert รายเมนูหลัง upsert menu_items ด้วย external_ref แล้ว: ถ้า item มี option_groups ให้ replace ทั้งชุด เฉพาะกลุ่มที่ source='storeos' ของเมนูนั้น (ลบ link+group เดิมที่มาจาก sync แล้วสร้างใหม่ตาม payload) — ง่ายและ idempotent กว่า diff รายตัว
3. สร้างกลุ่มinsert menu_option_groups (merchant_id, name, min_selection, max_selection, external_source='storeos') → insert menu_options (group_id, name, price=round(price), is_available) → insert menu_item_option_links (menu_item_id, option_group_id, sort_order ตามลำดับใน payload)
4. ไม่มี option_groupsitem ที่ไม่มี field นี้ (payload เวอร์ชันเก่า) หรือเป็น array ว่าง → ลบกลุ่ม source='storeos' ของเมนูนั้นทิ้ง (เมนูกลับเป็นเมนูเดี่ยว)
5. ราคาราคาที่ลูกค้าเห็น = menu_items.price + Σ option.price ที่เลือก (logic เดิมของ JDC ใช้ได้เลย)
ขาออเดอร์กลับมา (order.created): ไม่ต้องแก้อะไร — StoreOS รองรับ items[].options[{name,price}] อยู่แล้ว (เก็บลง order_items.modifiers และแสดงบนบอร์ด /delivery + ตั๋วครัว) ขอแค่ราคา item ใน payload รวมราคา option แล้ว ตามสัญญาเดิม

3) ผลกระทบ / สิ่งที่ไม่เปลี่ยน

  • HMAC, headers, การผูก merchant_id — เหมือนเดิมทุกอย่าง
  • ฝั่ง StoreOS นับ option_groups รวมใน sync hash แล้ว → แก้ variant/modifier ใน catalog จะ auto-push ขึ้น JDC ทันที (best-effort) + reconcile cron รายวันเก็บตก
  • JDC ที่ยังไม่อัปเดต Edge Fn: field ใหม่ถูกเมิน ไม่พังอะไร (backward compatible)
